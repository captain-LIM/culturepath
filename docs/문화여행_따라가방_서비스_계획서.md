## 1. 서비스 개요

### 1.1 한 줄 정의

“어떤 문화를 경험하고 싶은가”에서 출발해, 그 문화로 유명한 지역과 추천 관광지를 발견하고 나만의 코스로 엮는 취향 기반 문화여행 플랫폼.

### 1.2 핵심 컨셉

본 서비스의 진입 동선은 ‘문화 카테고리 → 유명 지역 → 추천 관광지 → 코스’ 순으로 설계한다. 기존 여행 앱이 ‘어디로 갈까(지역 우선)’를 묻는 것과 달리, 문화여행은 ‘어떤 문화를 경험할까(취향 우선)’를 먼저 묻는다. 사용자는 메인 화면에서 독립서점·문학·음악·전통주 같은 문화 카테고리 버튼을 선택하고, 해당 문화가 풍부한 지역이 데이터 기반으로 정렬되어 제시되며, 지역을 고르면 그 문화에 맞는 추천 관광지가 카드로 펼쳐진다.

이 흐름은 MZ세대의 여행 의사결정 방식(어디를 가느냐보다 나의 취향에 맞는 무엇을 경험하느냐)과 정확히 일치하며, 동시에 한국관광공사가 보유한 약 26만 건의 관광정보를 ‘문화 테마’라는 새로운 축으로 재구성해 활용한다는 데이터 활용 측면의 의의를 가진다.

### 1.3 해결하는 문제

- 취향 기반 탐색의 부재 — 기존 앱은 인기도·별점 기반 범용 정보에 머물러, 특정 문화 관심사로 지역과 장소를 엮어주는 진입점이 없다.

- 관광 수요의 집중 — 소수 대형 관광지에 수요가 쏠리는 반면, 소도시의 독립서점·갤러리·양조장 같은 마이크로 문화 거점은 발견 경로가 없다.

- 코스의 일회성 — 한 번 만든 코스가 공유·재사용되지 못하고 버려진다. (→ Fork·AI 변형으로 해결)

## 2. 핵심 사용자 흐름

서비스의 중심 동선은 다음과 같다. 메인에서 문화를 고르는 순간부터, 지역 발견과 관광지 추천이 한 줄기로 이어진다.

**메인(문화 카테고리 그리드) → 카테고리 선택(예: \#문학) → 유명 지역 리스트(통영·강릉…, 데이터 기반 정렬) → 지역 선택(예: 통영) → 추천 관광지 카드(박경리기념관 등) → 관광지 담기 / 코스 Fork / AI 변형 / 저장**

### 2.1 단계별 정의

| **단계** | **화면**      | **사용자 행동**            | **데이터 출처**           |
|----------|---------------|----------------------------|---------------------------|
| 1        | 메인          | 문화 카테고리 버튼 선택    | 내부 카테고리 정의        |
| 2        | 카테고리 상세 | 유명 지역 리스트 탐색·선택 | TourAPI 밀도 + 데이터랩   |
| 3        | 지역 상세     | 추천 관광지 확인·담기      | areaBasedList2 / detail\* |
| 4        | 관광지 상세   | 코스에 담기 또는 코스 Fork | places_cache / courses    |
| 5        | 코스 편집     | 순서 조정 · AI 변형 · 저장 | course_tracks / RAG       |

## 3. 화면 설계

### 3.1 메인 화면 — 문화 카테고리 그리드

서비스의 얼굴. 상단에 간단한 검색바와 함께, 문화 카테고리 버튼이 카드 그리드로 배치된다. 각 카테고리는 고유한 무드 컬러와 대표 이미지를 가져 ‘취향의 입구’처럼 느껴지도록 한다.

- 구성요소: 문화 카테고리 카드 그리드(2~3열), 시즌 추천 배너, 검색바, 하단 탭(홈·탐색·만들기·내정보).

- 동작: 카테고리 카드 탭 → 해당 문화의 유명 지역 리스트 화면으로 전환.

- 데이터: 카테고리는 내부 정의(아래 4장), 배너는 운영 큐레이션.

### 3.2 카테고리 상세 — 유명 지역 리스트

선택한 문화로 유명한 지역이 ‘왜 유명한지’와 함께 순위대로 제시된다. 예컨대 \#문학을 선택하면 통영(박경리·청마 유치환), 강릉(허균·허난설헌) 등이 관련 장소 밀도와 방문자 데이터를 결합한 점수 순으로 나열된다.

- 구성요소: 지역 카드 리스트(지역명, 대표 이미지, 한 줄 소개, 관련 장소 수, 문화 적합도 점수).

- 동작: 지역 카드 탭 → 지역 상세(추천 관광지) 화면. 지도 보기 토글 제공.

- 데이터: TourAPI 장소 밀도(areaBasedList2 + 분류코드) + 관광빅데이터(데이터랩) 방문자 추이로 정렬.

### 3.3 지역 상세 — 추천 관광지

선택한 ‘문화 × 지역’ 조합에 맞는 관광지가 카드로 큐레이션된다. 예: 통영 × 문학 → 박경리기념관, 청마문학관, 통영국제음악당 등. 각 카드에는 사진·운영시간·주소·간단 소개가 표시되고, ‘코스에 담기’ 버튼으로 즉시 일정에 추가할 수 있다.

- 구성요소: 추천 관광지 카드(사진·운영시간·주소·소개·담기 버튼), 이 지역의 인기 코스 섹션, 혼잡도 안내(P2).

- 동작: 관광지 담기 → 임시 코스에 추가 / 인기 코스 Fork → 편집 모드 진입.

- 데이터: areaBasedList2(지역+분류), detailCommon2·detailIntro2·detailImage2(상세), 연관 관광지 API(함께 가면 좋은 곳).

### 3.4 관광지·코스 상세 — 담기 · Fork · AI

추천 관광지를 모아 만든 코스는 음악 셋리스트(Track 1 → 2 → 3)처럼 시각화되어 동선의 흐름을 직관적으로 보여준다. 타인의 코스는 Fork 한 번으로 내 것으로 복제해 재구성하고, AI 어시스턴트에게 자연어로 변형을 요청할 수 있다.

- 구성요소: Track 타임라인, 동선 지도, Fork 버튼(원본 크레딧), AI 챗 진입.

- 동작: “당일치기로 압축”, “비 오는 날 실내 대안으로” 등 자연어 요청 → 변형 결과 미리보기 → 적용.

## 4. 문화 카테고리 정의 및 지역 매핑

### 4.1 문화 카테고리 (1차 10종)

초기 카테고리는 마이크로 문화 거점이 분명하고 TourAPI 분류코드로 식별 가능한 10종으로 시작하며, 신분류체계 코드(lclsSystm) 재조합으로 점진 확장한다.

| **문화 카테고리** | **설명**                | **대표 지역**  | **추천 관광지 예시**     |
|-------------------|-------------------------|----------------|--------------------------|
| 독립서점·책방     | 동네 책방·북스테이 감성 | 강릉·전주·서울 | 독립서점 거리, 북스테이  |
| 문학              | 작가의 흔적·문학관      | 통영·강릉·하동 | 박경리기념관, 청마문학관 |
| 음악              | 공연장·음악 거점        | 통영·춘천      | 통영국제음악당           |
| 전통주·양조장     | 지역 양조장·전통주      | 전주·안동      | 전통주 공방·양조장 투어  |
| 로컬 미식         | 지역 고유 식문화        | 전국 소도시    | 전통시장·로컬 맛집       |
| 공예·공방         | 수공예·체험 공방        | 전주·통영      | 한옥마을 공방거리        |
| 근대 문화유산     | 근대 건축·산업유산      | 포항·군산·목포 | 근대문화거리·산업유산    |
| 미술·갤러리       | 소규모 갤러리·아트씬    | 포항·서울      | 로컬 갤러리              |
| 영화·애니메이션   | 애니·만화·영화 감성     | 춘천           | 애니메이션박물관         |
| 커피·카페         | 커피 문화 거점          | 강릉           | 안목해변 커피거리        |

*※ 추천 관광지 예시는 큐레이션 시드이며, 실제 노출 목록은 TourAPI 데이터로 자동 산출·검증한다.*

### 4.2 ‘유명한 지역’ 산출 로직

특정 문화로 ‘유명한 지역’은 다음 점수로 정렬한다. 초기에는 실제 사용자 활동 데이터가 없으므로 활동량을 0점으로 넣지 않고 큐레이션 비중으로 대체한다.

**초기 지역 문화점수 = (장소 밀도 × 0.4) + (데이터랩 방문자 추이 × 0.3) + (큐레이션 가중치 × 0.3)**

사용자 활동 데이터가 충분히 쌓이면 다음 장기 점수로 전환한다.

**장기 지역 문화점수 = (해당 카테고리 장소 밀도 × 0.4) + (데이터랩 방문자 추이 × 0.3) + (사용자 코스·Fork 활동량 × 0.2) + (큐레이션 가중치 × 0.1)**

- 장소 밀도: 장기적으로 areaBasedList2 + 콘텐츠타입·신분류체계 코드로 지역별 해당 문화 장소 수를 집계한다. R4 초기 구현은 기존 지역 카드의 `spotCount`를 큐레이션 입력으로 사용하며 실제 TourAPI 전수 집계값이 아니다.

- 방문자 추이: 관광빅데이터(데이터랩)로 ‘요즘 뜨는 소도시’ 가산점, 외지인 비율로 신흥 거점 발굴.

- 사용자 활동량: 해당 지역·카테고리 코스 생성·Fork·완주 데이터(서비스 성장 시 비중 확대).

## 7. 백엔드 및 데이터베이스 설계

### 7.1 아키텍처

Flutter 클라이언트 ↔ Node.js/Express REST API ↔ (MySQL 코어 DB · Redis 캐시/랭킹 · Qdrant RAG 벡터 · TourAPI 외부 · OpenRouter LLM). TourAPI 응답은 places_cache에 적재해 호출 횟수 제한에 대응하고 코스 데이터와 조인한다.

### 7.2 주요 API

| **도메인** | **엔드포인트**                                | **설명**                          |
|------------|-----------------------------------------------|-----------------------------------|
| 문화       | GET /cultures                                 | 문화 카테고리 목록                |
| 문화       | GET /cultures/:id/regions                     | 해당 문화로 유명한 지역 정렬 목록 |
| 지역       | GET /regions/:code/spots?culture=             | 지역×문화 추천 관광지             |
| 장소       | GET /places/search · /:id · /:id/related      | 검색·상세·연관(TourAPI 프록시)    |
| 코스       | POST /courses · PUT /courses/:id              | 코스 생성·수정                    |
| 코스       | POST /courses/:id/fork                        | Fork(복제 + 원본 크레딧)          |
| 코스       | GET /courses/feed · /ranking                  | 피드·랭킹                         |
| 완주       | POST /courses/:id/checkin                     | GPS 완주 인증                     |
| AI         | POST /ai/transform                            | RAG 코스 변형                     |
| 인증       | POST /auth/register · /login · /migrate-guest | 회원·게스트 이관                  |

## 8. 프론트엔드 설계 (Flutter)

단일 코드베이스로 Android·iOS·Web 동시 대응. 상태관리는 Riverpod, 라우팅은 go_router(웹 URL·딥링크 지원). 게스트 로컬 저장(SharedPreferences) 후 로그인 시 서버 마이그레이션.

### 8.1 화면 구조

- 홈: 문화 카테고리 그리드 + 시즌 배너 + 검색바.

- 카테고리 상세: 유명 지역 리스트(문화 적합도 점수 표시) + 지도 토글.

- 지역 상세: 추천 관광지 카드 + 이 지역 인기 코스.

- 코스 빌더: 드래그앤드롭 보드 + 장소 검색 바텀시트.

- 코스 상세: Track 타임라인 + 동선 지도 + Fork + AI 챗.

- 프로필: 내 코스 · Fork한 코스 · 완주 스탬프.

### 8.2 핵심 컴포넌트

| **컴포넌트**       | **역할**                                   |
|--------------------|--------------------------------------------|
| CultureGrid        | 메인 문화 카테고리 카드 그리드             |
| RegionRankList     | 문화별 유명 지역 정렬 리스트               |
| SpotCard           | 추천 관광지 카드(담기 버튼 포함)           |
| CourseBuilderBoard | ReorderableListView 드래그앤드롭 일정 편집 |
| TrackTimeline      | Track 1→2→3 셋리스트 타임라인              |
| AiChatView         | 자연어 코스 변형 + diff 미리보기           |

## 10. 추천 · 확장 기능

핵심 동선을 단단히 만든 위에, 사용자 순환·바이럴·지속성을 높이는 기능을 단계적으로 붙인다.

| **기능**               | **가치**                                  | **난이도** | **권장 시점** |
|------------------------|-------------------------------------------|------------|---------------|
| 완주 인증·스탬프       | 랭킹 신뢰도 + 재방문 동기(게이미피케이션) | 중         | P1            |
| 코스 공유 링크·딥링크  | 카톡·인스타 바이럴 유입 채널              | 하         | P2            |
| 예상 비용·시간 추정    | 여행 의사결정 실질 정보                   | 하~중      | P2            |
| 코스 컬렉션(폴더)      | 북마크 정리·재사용성                      | 하         | P2            |
| 시즌·이벤트 테마       | 계절성으로 재방문·바이럴                  | 하         | P2            |
| 작성자 통계 대시보드   | 창작 동기 + 데이터 환원 어필              | 중         | P2            |
| 큐레이터 팔로우        | 창작자 생태계 형성                        | 중         | P2            |
| 오프라인 코스 다운로드 | 현장 사용성(신호 불안정 대응)             | 중~상      | 확장          |
| 개인화 추천(취향 벡터) | 성장할수록 정밀해지는 추천                | 상         | 장기          |
| 소상공인 제휴·쿠폰     | 지역경제 선순환·수익 모델                 | 상         | 사업화        |

**강력 추천 —** 완주 인증·스탬프는 랭킹 알고리즘의 입력값인 ‘완주율’을 실제로 수집할 유일한 수단이므로 P1로 끌어올린다. 또한 공유 링크는 난이도 대비 유입 효과가 커 우선 확장 1순위로 권장한다.
