# [황찬우 전용] 잔여 PR 로드맵과 세션 인수인계

> **문서 소유자:** 황찬우
>
> **최종 갱신:** 2026-07-23
>
> **담당 범위:** 외부 API 연동 · RAG/AI Backend · Frontend UI/UX Design 및 API 연결
>
> **목적:** 대화 세션이 바뀌어도 완료 상태, 다음 PR 범위, 의사결정, 검증 기준을 동일하게 이어가기 위한 기준 문서

## 1. 이 문서의 지위

- 이 문서는 황찬우 담당 작업을 **PR 실행 단위**로 나눈 기준 문서다.
- 상세 요구사항은 [황찬우 전용 실행계획](./HWANG_CHANWOO_WORK_PLAN.md), 외부 API의 실제 검증 이력은 [외부 관광 API 검증 체크리스트](./API_VALIDATION_CHECKLIST.md), 장소 필드 계약은 [TourAPI 장소 계약](./TOUR_PLACE_CONTRACT.md)을 함께 읽는다.
- 아직 생성되지 않은 PR은 GitHub 번호 대신 `R1`부터 `R10`까지 로드맵 ID로 식별한다.
- PR을 머지하면 해당 항목에 실제 PR 번호·링크·완료일을 기록하고 다음 미완료 항목을 현재 작업으로 지정한다.
- 문서보다 현재 코드나 공식 외부 API 명세가 더 최신이면 읽기 전용으로 차이를 확인한 뒤, 구현 전에 사용자에게 영향과 선택지를 보고한다.
- 외부 API 키, Qdrant 키, OpenRouter 키, 인증 헤더와 키가 포함된 전체 URL은 이 문서에 기록하지 않는다.

## 2. 세션이 바뀌었을 때 반드시 지킬 절차

새 구현 세션은 다음 순서를 따른다.

1. `git status --short --branch`와 최근 커밋을 확인한다.
2. 저장소 루트의 로컬 `AGENTS.md`와 `.agents/WORKFLOW.md`를 읽는다. 이 파일들은 Git에 올라가지 않는 황찬우 전용 작업 규칙이다.
3. 이 문서와 현재 PR에 연결된 상세 문서를 읽는다.
4. 첫 번째 미완료 PR의 목표, 포함·제외 범위, 예상 변경 파일, 테스트, 완료 조건을 사용자에게 설명한다.
5. `결정 필요` 항목마다 선택지, 장단점과 권장안을 제시한다.
6. 사용자가 선택을 완료해도 바로 구현하지 않는다. 사용자가 별도 메시지로 명시적으로 `구현 시작`을 지시한 뒤 브랜치를 만들고 코드를 수정한다.
7. 구현 후 로컬 테스트와 diff 검사를 실행한다.
8. 구현에 참여하지 않은 별도 리뷰 에이전트를 `gpt-5.6-sol`, reasoning effort `high`로 명시해 생성한다.
9. 리뷰 판정과 심각도별 발견 사항, 수정 결과, 테스트, 잔여 위험을 사용자에게 공개한다.
10. 커밋·푸시·PR 생성은 사용자의 별도 요청이 있을 때만 수행한다.

범위가 커지거나 임수민 담당의 공개 API·DB·Flutter 계약에 영향을 주는 새 변경이 발견되면 구현을 멈추고 추가 결정을 받는다.

### 3.2 아직 실데이터로 연결되지 않은 지점

- `backend/src/controllers/placesController.js`는 아직 `ALL_PLACES` 하드코딩 배열을 사용한다.
- 공개 `/places/search`, `/places/:id`, `/places/:id/related`는 새 TourAPI 서비스 계층을 호출하지 않는다.
- `regionsController.js`의 문화별 지역·장소 데이터도 시드 상태다.
- 목록 단계에서 얻을 수 없는 개요, 운영시간, 휴무일과 추가 이미지는 아직 `null`이다.
- Swagger에는 새 TourAPI 기반 공개 장소 계약이 아직 추가되지 않았다.
- MySQL 캐시, Qdrant, OpenRouter와 구조화된 `/ai/transform`은 아직 완성되지 않았다.
- Flutter는 현재 공개 API의 기존 응답 형태와 Mock/시드 데이터에 의존하는 부분이 있다.

### 3.3 실제 API 검증 시 주의할 현재 사실

- `lclsSystmCode2` 실제 호출은 성공했다.
- `areaBasedList2`와 `searchKeyword2`의 기존 실제 호출은 폐기 예정인 `areaCode`·`sigunguCode` 호환 파라미터로 수행됐다.
- 현재 코드는 공식 현행 계약인 `lDongRegnCd`·`lDongSignguCd`를 사용하지만, 이 조합의 live 호출은 아직 하지 않았다.
- 다음 승인된 smoke test에서 `ldongCode2`와 현행 법정동 기반 목록·검색을 확인해야 한다.
- `detailCommon2`, `detailIntro2`, `detailInfo2`, `detailImage2`는 원본 API 동작을 이미 확인했지만 서비스 계층과 공개 API 조립은 아직 구현하지 않았다.
- 기본 테스트는 `.env` 키를 비우고 외부 네트워크를 차단한다. live 호출은 사용자가 승인한 횟수와 엔드포인트에서만 한다.

## 5. R1 — TourAPI 상세조회와 공개 장소 API 수직 연결

### 목표

하드코딩 장소 응답을 TourAPI 서비스로 교체해 Backend의 장소 검색과 상세조회가 실제 관광 데이터를 반환하게 한다.

### 확정된 결정

- 기존 Flutter 호환 배열 응답과 필수 문자열 필드 유지
- `q` 검색과 법정동 지역 목록을 하나의 공개 검색 API에서 분기
- 이번 PR에서는 현행 법정동 코드를 직접 입력하고 내부 지역 slug 매핑은 연기
- 공통·소개·이미지를 기본 상세로 조립하고 반복 상세는 선택 메서드만 제공
- 원인별 구조화 오류와 HTTP 상태 사용
- CulturePath 자체 Swagger UI와 OpenAPI JSON 제공
- 서비스 주입형 컨트롤러 단위 테스트 사용
- live smoke test 최대 4회, 재시도 없음

### 포함 범위

- `ldongCode2` 서비스 메서드와 법정동 코드 응답 정규화
- `detailCommon2`, `detailIntro2`, `detailInfo2`, `detailImage2` 서비스 메서드
- 목록 `PlaceSummary`와 상세 응답을 결합하는 내부 `PlaceDetail` 계약
- 공개 `GET /places/search`와 `GET /places/:id`의 TourAPI 연결
- 입력 검증, 빈 결과, 원본 업무 오류, 타임아웃의 공개 오류 계약
- 페이지네이션과 nullable 상세 필드 처리
- Swagger/OpenAPI 명세와 관련 `docs/` 갱신
- fixture/mock 기반 정상·빈 결과·오류·단일 item·이미지 없음 테스트
- 사용자가 승인한 최소 live smoke test

### 제외 범위

- MySQL 캐시와 stale fallback
- `/places/:id/related` 실데이터 전환
- DataLab 지역 점수
- Flutter 코드 변경
- Qdrant와 OpenRouter

### 결정 필요

1. `/places/search` 응답을 기존 배열로 유지할지, `items`와 페이지 메타데이터를 가진 객체로 변경할지
2. 첫 상세 응답에 공통·소개·이미지만 포함할지, 반복 상세정보까지 포함할지
3. Flutter가 법정동 코드를 직접 전달할지, Backend가 기존 앱 지역 코드를 법정동 코드로 변환할지
4. 캐시가 없는 단계에서 TourAPI 장애를 어떤 HTTP 상태와 오류 body로 반환할지
5. live smoke test 대상과 최대 호출 횟수

이 선택은 임수민이 구현한 Flutter 파서와 공개 API 계약에 영향을 줄 수 있으므로 구현 전에 영향 범위를 명시한다.

### 완료 조건

- `/places/search`가 하드코딩 배열이 아니라 TourAPI 기반 결과를 반환한다.
- `/places/:id`가 공통·소개·이미지 정보를 일관된 내부 모델로 반환한다.
- Swagger와 실제 응답이 일치한다.
- 비밀값과 전체 요청 URL이 로그·오류·diff에 없다.
- 기본 테스트가 외부 호출 없이 통과한다.
- 승인된 경우 현행 법정동 파라미터 live 호출 결과가 검증 문서에 기록된다.

## 6. R2 — MySQL 장소 캐시와 장애 fallback

### 목표

TourAPI 호출 비용과 지연을 줄이고 외부 장애 중에도 최근 장소 데이터를 제공한다.

### 포함 범위

- 임수민과 `places_cache` 스키마·마이그레이션 소유권 합의
- `contentId` 기준 upsert와 원본 갱신 시각·캐시 시각 분리
- 목록·상세 필드, 분류, 이미지 저장 정책
- TTL과 명시적 refresh 정책
- 동일 키 동시 요청 중복 방지
- fresh hit, miss, expired refresh, stale fallback 테스트
- 캐시 출처와 갱신 상태를 내부에서 관찰할 수 있는 최소 로그

### 제외 범위

- 전국 데이터 일괄 수집
- Qdrant 인덱싱
- 연관 관광지와 DataLab 원본 적재

### 결정 필요

- 테이블 생성과 마이그레이션 담당자
- JSON 컬럼과 정규 컬럼의 경계
- 기본 TTL, stale 허용 기간, 수동 갱신 권한
- 캐시 장애 시 외부 API 직통 허용 여부

### 완료 조건

- 동일 조건 반복 요청이 불필요한 외부 호출을 만들지 않는다.
- 외부 API 장애 시 정책에 맞는 stale 데이터 또는 명확한 오류가 반환된다.
- 캐시 갱신 실패가 기존 정상 데이터를 삭제하지 않는다.

## 16. 공통 테스트·리뷰·보안 기준

모든 코드 PR은 다음을 만족한다.

- 정상, 빈 결과, 잘못된 입력, 외부 업무 오류, 타임아웃을 범위에 맞게 테스트한다.
- 외부 응답의 `items.item`이 배열, 단일 객체, 빈 문자열 또는 누락인 경우를 처리한다.
- 기본 테스트는 실제 외부 API와 유료 서비스에 접속하지 않는다.
- live smoke test는 사용자 승인 후 최소 횟수만 실행한다.
- `serviceKey`, Qdrant 키, OpenRouter 키와 전체 인증 URL을 출력하지 않는다.
- 현재 환경에서는 `.agents/scripts/verify.cmd` 또는 범위별 명령으로 검증한다.
- 별도 `gpt-5.6-sol`·`high` 리뷰 에이전트가 실제 diff와 테스트를 확인한다.
- 공개 API나 Flutter 모델이 바뀌면 Swagger, 관련 `docs/`, Flutter 파서를 같은 계약으로 맞춘다.
- 임수민 담당 파일과 겹치면 구현 전에 변경 파일과 호환성 영향을 알린다.

## 17. 로드맵 갱신 규칙

각 PR이 머지될 때 다음을 갱신한다.

1. 해당 로드맵 ID의 상태를 `머지 완료`로 변경한다.
2. 실제 PR 번호, 링크, 머지 날짜를 추가한다.
3. 구현 중 확정된 계약과 남은 위험을 해당 항목에 반영한다.
4. 테스트 개수는 참고값으로만 기록하고 새 세션에서 다시 측정한다.
5. 새로 생긴 필수 작업은 기존 PR에 억지로 넣지 않고 별도 ID를 추가한다.
6. 다음 미완료 항목을 `다음 작업·계획 필요`로 표시한다.

## 18. 현재 세션 인수인계 포인트

- 현재 작업은 **R1 — TourAPI 상세조회와 공개 장소 API 수직 연결**이다.
- 사용자는 R1의 모든 결정사항을 A안으로 승인한 뒤 별도로 구현 시작을 지시했다.
- 구현과 테스트를 완료했고, 독립 리뷰의 Medium·Low 발견 사항을 모두 반영한 뒤 재리뷰 `APPROVE`를 받았다.
- 승인된 live 호출 4회는 모두 사용했다. `areaBasedList2`와 `searchKeyword2`는 성공했지만 `ldongCode2?lDongListYn=N`은 타임아웃으로 live 재검증이 남았다. 새 승인 없이 추가 호출하지 않는다.
- 최종 검증이 끝나도 사용자가 별도로 요청하기 전에는 커밋·푸시·PR을 생성하지 않는다.
- R1이 머지되면 이 표를 갱신하고, 다음 PR은 R2~R5 의존성과 원격 최신 상태를 다시 확인한 뒤 같은 승인 게이트로 계획한다.
