# Backend tests

이 디렉터리의 테스트는 팀과 공유하며 Git에 포함한다.

## 원칙

- 기본 테스트에서는 실제 TourAPI, 관광 빅데이터 API, Qdrant, OpenRouter를 호출하지 않는다.
- 외부 응답과 비밀 값은 제거한 fixture와 mock으로 재현한다.
- 구현 변경에는 정상 흐름뿐 아니라 빈 결과와 오류 흐름 테스트를 포함한다.
- 로컬 실제 API smoke test는 `.agents/` 아래의 로컬 전용 도구로 분리한다.
- 기본 테스트 부트스트랩은 외부 API 및 DB 환경 변수를 빈 값으로 고정한다.
- 기본 테스트는 `fetch`, HTTP/HTTPS, TCP/TLS 연결을 차단한다.
- 기본 테스트가 0개이면 로컬 검증 스크립트가 실패해야 한다.

## 우선 작성할 테스트

1. 공공데이터 정상 응답 정규화
2. 최상위 `resultCode` 오류 객체 변환
3. `items.item` 배열·단일 객체 처리
4. `items=""`와 누락된 `items` 처리
5. 인증키 마스킹과 이중 인코딩 방지
6. 외부 요청 타임아웃
7. TourAPI 장소 데이터를 내부 모델로 변환

테스트 파일은 `*.test.js` 형식을 사용하고 Node.js 내장 `node:test`를 기본 러너로 사용한다.
